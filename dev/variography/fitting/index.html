Fitting variograms

Overview

Fitting theoretical variograms to empirical observations is an important modeling step to ensure valid mathematical models of spatial continuity. Given an empirical variogram, the fit function can be used to perform the fit:

StatsBase.fit — Method
fit(V, γ, [algo])

Fit theoretical variogram type V to empirical variogram γ using algorithm algo. Default algorithm is WeightedLeastSquares.

Examples

julia> fit(SphericalVariogram, γ)
fit(Variogram, γ, [algo])

Fit all subtypes of Variogram to empirical variogram γ and return the one with minimum error as defined by the algorithm algo.

Examples

julia> fit(Variogram, γ)

Example

In this example we generate data from a known theoretical model and fit other models to the data. First, we select a single model and evaluate it at different lags to create a synthetic empirical variogram:

# create data from a model
γ = SphericalVariogram()
x = range(0, stop=3, length=20)
y = γ.(x)
n = 20:-1:1

# synthetic empirical variogram
g = EmpiricalVariogram(x, y, n)

plot(g)

We can then fit different models to the empirical variogram:

γs = [fit(V, g) for V in [GaussianVariogram, ExponentialVariogram, MaternVariogram]]

# plot all models
p = plot(g)
for γ in γs
  plot!(γ, maxlag=3.)
end
p

Alternatively, we can let GeoStats.jl find the model with minimum error:

γₒ = fit(Variogram, g)
SphericalVariogram{Float64,Distances.Euclidean}
  range: Float64 1.0000000092581833
  sill: Float64 0.9999999932063643
  nugget: Float64 2.2150606971757795e-8
  distance: Distances.Euclidean

Methods

Weighted least squares

Variography.WeightedLeastSquares — Type
WeightedLeastSquares()
WeightedLeastSquares(w)

Fit theoretical variogram using weighted least squares with weighting function w (e.g. h -> 1/h). If no weighting function is provided, bin counts of empirical variogram are normalized and used as weights.
